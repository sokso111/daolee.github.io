Python Coding Style

每种编程语言可能使用不同的风格，其中一些比另一些可读性更强。让别人能更容易的阅读你的代码通常都会是个不错的主意，采用一个很好的编码风格将有助于此。

对于Python，PEP 8已经成为大多数项目遵循的风格指南;它促成了一种可读性很高并且令人愉悦的代码风格。每个Pythoner都应该在了解它，下面是提取其中的比较重要的几点：

  • 使用4个空格而不是制表符表示缩进

  • 4个空格是小缩进之间的一个很好的妥协（允许更大的嵌套深度）和大的缩进（可读性更高）。最好不使用制表符是因为其更容易引起混乱。

  • Wrap lines不超过79个字符

  • 这可以帮助用户使用小显示器，并且在大显示器上并排放置多个代码文件。

  • 使用空行来分离函数和类，以及更大的代码块。

  • 可能的话，尽量将注释放在单独的一行。

  • 使用docstrings

  • 在运算符周围或者逗号后面使用空格，而不是直接在包围结构内部：a = f(1, 2) + g(3, 4).

  • 按惯例命名你的类和函数; 一般是使用CamelCase命名类和lower_case_with_underscore命名函数和方法。始终使用self作为第一个方法参数的名称 (详情参见A First Look at Classes)。

  • 如果你的代码旨在在国际环境中使用，请勿使用花式编码。 在任何情况下，Python的默认值，UTF-8或者纯ASCII工作效果更好。

  • 同样的，如果讲不同语言的人来阅读或者维护代码的几率非常小，请不要在标识符中使用非ASCII字符。
